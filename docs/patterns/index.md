# Patterns

Reusable moves and judgement-affording Distinctions for evolving stride itself. Different from [Reference](/reference/agentic-engineering) (static lookups) and [Skills](/skills/vision) (commands you run) — these are the moves you reach for when stride's own discipline meets a hard question.

| Entry | Kind | When it applies |
|:--|:--|:--|
| [Work on the project, in the project](./work-on-and-in) | Context | Using stride for real work surfaces a flaw in stride — the friction is the highest-signal source of what to fix next |
| [Friction is information. The question is which kind.](./friction-distinction) | Distinction | A user hits stride friction — is it signal (push back on the work) or context-mismatch (calibrate the gate)? |
| [New issue, not new scope](./new-issue-not-new-scope) | Pattern | Work surfaces mid-flight that wasn't in scope — file it as a new issue rather than absorbing it into the current one |
| [Revise, don't stretch](./revise-dont-stretch) | Pattern | The Vision-trace feels strained at `/linear:plan-work` draft time |
| [Vision evolves with the work](./vision-evolves-with-the-work) | Pattern | A trace gate fires because no Vision criterion fits — add the missing criterion on the same branch, not in a follow-up PR |
